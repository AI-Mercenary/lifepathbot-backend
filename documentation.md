# LifePathBot Platform Documentation

LifePathBot is an AI-powered educational and peer-mentorship ecosystem designed for students. It contains a robust React-based frontend hosted on Vercel and an Express Node.js backend hosted on Leapcell, utilizing the ultra-fast Groq API for AI processing.

## 📌 1. Project Architecture
- **Frontend Repository**: `AI-Mercenary/lifepathbot-frontend` (Local: `e:\lifepathbot-frontend`)
- **Backend Repository**: `AI-Mercenary/lifepathbot-backend` (Local: `e:\lifepathbot-backend`)
- **Database**: MongoDB Atlas 
- **AI Engine**: Groq API (Model: `llama-3.3-70b-versatile`)
- **Authentication**: Firebase Auth (Google Sign-In + Email/Password)

## 📌 2. Core Features & Modes
### A. The AI Chatbot
The system is built on a dual-mode interaction architecture:
1. **Study Mode**: Students can upload their course PDFs. The backend parses the PDF into memory (RAG) and explicitly answers questions referencing the uploaded materials.
2. **General Mode**: Operates as a peer mentor, using system prompts that rely on the Community Hub suggestions to give students campus-specific, peer-approved guidance.
*Note: All references to "Ollama" or "Llama" have been completely anonymized to present the bot as a white-labeled assistant to students.*

### B. Suggestion Platform (Community Hub)
A space where seniors and peers can drop advice for juniors.
- **Fields Tracked**: Author Name, Branch/Department, Year of Study, Category (Domain), and Detailed Guidance.
- **Moderation**: All student-submitted suggestions default to a `pending` status. They must be explicitly approved from the Admin Dashboard before appearing publicly.

### C. The Admin Dashboard
An exclusive portal for managing the platform.
- **User Management**: View all registered students, their IDs, departments, and roles.
- **Chat Logs**: Monitor all AI interactions across the university to maintain safety and understand student needs.
- **Content Moderation**: Approve or reject suggestions and uploaded community content.
- **Bulk CSV Upload**: Admins can import `.csv` or `.xlsx` files generated by Google Forms. The parser intelligently reads specific columns (e.g., "What type of guidance do you want to give...") and automatically pipes them into the database.

## 📌 3. API Integrations & Data Flow
### Backend Routes
- `/api/chat`: Handles history retrieval and user chat sessions.
- `/api/chat/ask`: Accepts the prompt and RAG context, directly interfacing with the Groq API for lightning-fast completions.
- `/api/chat/upload-material`: Parses PDF files dynamically via memory buffers (bypassing local disk storage) and creates temporary prompt injection text.
- `/api/suggestions`: Returns approved community suggestions or handles bulk creation.
- `/api/users/sync`: Secures Firebase login verification and syncs the Google Authentication user ID across the MongoDB collections.

## 📌 4. Deployment Workflow

### Frontend (Vercel)
The frontend dynamically communicates with the backend via the `VITE_API_URL` environment variable. 
**To redeploy:**
1. Push any local code changes to `lifepathbot-frontend`'s GitHub repository.
2. Vercel automatically detects the new commit and rebuilds the React ecosystem.

### Backend (Leapcell)
The backend requires robust CORS configuration and proper environment variables to operate.
**Environment Variables Required:**
- `MONGO_URI`: Access link to the MongoDB database cluster.
- `GROQ_API_KEY`: The authentication key for the AI model processing.
- `PORT`: (Managed dynamically by Leapcell).

**To redeploy:**
1. Run `git add .`, `git commit -m "Your message"`, and `git push origin main` in the `lifepathbot-backend` folder.
2. Wait for the code to hit GitHub.
3. On the Leapcell Dashboard, hit the **redeploy** button to fetch the latest commit. Avoid pushing files larger than 50MB (like executables or database dumps) to keep the CI/CD pipeline fast.

## 📌 5. Troubleshooting Common Issues
- **AI connection failed**: This means the `GROQ_API_KEY` isn't accessible, or the backend container crashed instantly on startup. Check Leapcell logs. If `MONGO_URI` is wrong, the server prints a JSON error instead of dropping the connection layout.
- **CORS blocked by origin**: Indicates the Leapcell target deployment is either asleep, crashed, or missing preflight settings. The backend explicitly returns `.setHeader('Access-Control-Allow-Origin', ...)` dynamically, so CORS errors are exclusively a symptom of deployment crashes or timeout errors!
- **Bulk Upload missing content**: Make sure the CSV explicitly possesses the columns defined in `AdminDashboard.tsx`, such as `Department`, `Year of Study`, and `Detailed Suggestion`.
